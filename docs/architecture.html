Project: Quill

How the finished product works on iPad and Windows

Same web codebase, two delivery shapes, one synced data store.

01The three pieces

Your codebase ships in three forms. Two of them are apps you interact with; one is a small server you host. All three speak the same data protocol so notes, folders, and pages stay identical everywhere.

iPad

Installed as a PWA

Open the URL in Safari. Tap Share → "Add to Home Screen". Now it lives on your home screen with a real app icon and fullscreen launch — no App Store, no Mac, no Apple Developer fee.

  • Apple Pencil handwriting with pressure and tilt
  • Works offline; changes queue and sync on reconnect
  • Notes stored in IndexedDB locally
  • Looks and feels like a native app once installed
delivered via your URL
WebSocket
Server

Sync relay you host

A tiny Node program (y-websocket-server). Runs on a VPS, a Raspberry Pi, or anything that can run Docker. Stateless or with persistence — your choice.

  • Relays Y.js updates between connected clients
  • Helps late-joining devices catch up to current state
  • Optional Postgres backing for durable history
  • Auth layer you control (JWT, room invites, etc.)
your domain · your control
WebSocket
Windows

Native app via Tauri

The same web app, wrapped in a Tauri 2 native shell. Compiled into a real Windows .exe installer. Shows up in Start menu, taskbar, system tray — like any native program.

  • Type fluidly with a real rich-text editor (Tiptap)
  • View handwritten notes from your iPad as you make them
  • Native menu bar, file system, drag-and-drop, OS notifications
  • Built on Windows (no Mac needed); Linux build follows

02What happens when you draw a stroke on iPad

Sketch of the data path from Apple Pencil tip to the Windows screen, with realistic timings for a healthy network.

  1. 1

    Apple Pencil draws

    Safari fires pointerdown/pointermove events with position, pressure, and tilt. We capture coalesced events so no points are dropped.

    ~0ms
  2. 2

    Ink renders locally

    The stroke draws to canvas immediately via perfect-freehand — outline polygon with pressure-based width. Feels like actual ink to the user.

    ~10ms
  3. 3

    Stroke saved to IndexedDB

    On stroke completion, the local Y.js document gets ystrokes.push([stroke]). Persisted via y-indexeddb — survives reload, works offline.

    ~5ms
  4. 4

    Y.js sends the update over WebSocket

    The diff is encoded as a tiny binary payload (usually a few hundred bytes per stroke) and sent to your sync server.

    ~50–150ms
  5. 5

    Server broadcasts to other peers

    y-websocket-server forwards the update to every connected device subscribed to the same document.

    ~10ms
  6. 6

    Windows receives and applies

    Tauri app's embedded Y.js applies the update. The ystrokes.observe() callback fires; the canvas re-renders with the new stroke.

    ~30ms
Total pen-to-Windows-screen ~100–500ms typical

03Stack at a glance

One web codebase. Two delivery shapes. One sync server. Open-source from top to bottom.

UI framework React or Sveltefinal choice during Phase 1 — both work
Ink rendering perfect-freehand (Steve Ruiz, MIT)drop-in for our prototype's outline algorithm
Rich text editor Tiptap or Lexicalfor typed content alongside ink
Sync data model Y.js (CRDT) — Y.Array of strokes per page, Y.Map for metadata
Sync transport y-websocket — connects browser to your server over WebSocket
Sync server y-websocket-server in Node~50 lines of config, optional Postgres for history
Local persistence y-indexeddb on web, Tauri file system on desktop
iPad delivery PWA — open URL in Safari, "Add to Home Screen", done
Desktop delivery Tauri 2 — wraps the web app into a native .exe / Linux binary
Hosting Static web app on any CDN; sync server on any VPS — all yours to control
License AGPL-3.0 recommended for FOSSforces service operators to share modifications

The result: on iPad, you tap an app icon on your home screen and write with the Pencil. On Windows, you double-click an icon on the desktop and type around the strokes that just appeared. Same notes, same folders, same data — flowing through a server you control, with no App Store, no Mac, and no platform fees.
